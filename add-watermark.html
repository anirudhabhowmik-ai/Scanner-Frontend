    🔧 How Watermark Positioning Works (Three Coordinate Systems)

    The hardest part of this tool was converting between three different coordinate systems. Here's how it works:

    Step 1: Frontend Pixel Coordinates 

    // During drag, we track pixel positions relative to card
const startLeft = parseFloat(el.style.left) || 0;
const startTop = parseFloat(el.style.top) || 0;
const startX = e.clientX;
const startY = e.clientY;

// Calculate new position with delta
const deltaX = ev.clientX - startX;
const deltaY = ev.clientY - startY;
let newLeft = startLeft + deltaX;
let newTop = startTop + deltaY;

    The browser gives us pixel positions relative to the viewport. We convert to positions relative to the page card.

    Step 2: Convert to Canvas-Relative Percentages 

    // Calculate position relative to canvas
const canvasRect = canvas.getBoundingClientRect();
const cardRect = card.getBoundingClientRect();
const canvasOffsetLeft = canvasRect.left - cardRect.left;
const canvasOffsetTop = canvasRect.top - cardRect.top;

const canvasRelativeX = cardRelativeLeft - canvasOffsetLeft;
const canvasRelativeY = cardRelativeTop - canvasOffsetTop;

// Convert to percentages
const leftPercent = (canvasRelativeX / canvasWidth) * 100;
const topPercent = ((canvasHeight - canvasRelativeY) / canvasHeight) * 100;
// Flip Y so 0% = top, 100% = bottom (matches user expectation)

    Step 3: Backend PDF Points 

    # Convert percentages to PDF points (1 point = 1/72 inch)
x_percent = float(left_val.rstrip('%'))
y_percent = float(top_val.rstrip('%'))

x = (x_percent / 100) * width  # width in points
y = height - ((y_percent / 100) * height)  # Flip Y for PDF's bottom-left origin

    Step 4: ReportLab Transformation 

    # ReportLab uses its own coordinate system with transformations
c.saveState()
c.translate(x, y)        # Move to center point
c.rotate(rotation_deg)   # Apply rotation

if wm_type == "text":
    c.drawCentredString(0, -text_height/3, text)  # Centered at translate point
elif wm_type == "image":
    c.drawImage(ImageReader(img_buffer), -img_width/2, -img_height/2, width=img_width)

    ReportLab's translate moves the origin to our desired center point, then we draw the watermark centered at (0,0) with rotation.

    function makeResizable(el, img, handle, position, card) {
    handle.addEventListener("pointerdown", (e) => {
        const startWidth = img.offsetWidth;
        const startX = e.clientX;
        const aspectRatio = img.offsetHeight / startWidth;
        
        function move(ev) {
            let diffX = ev.clientX - startX;
            let diff = position === "br" || position === "tr" ? diffX : -diffX;
            
            let newBaseWidth = baseWidth + diff / visualScale;
            newBaseWidth = Math.max(50, Math.min(400, newBaseWidth));
            
            img.style.width = newBaseWidth * visualScale + "px";
            img.style.height = newBaseWidth * aspectRatio + "px";
        }
    });
}

    🐛 The Debugging Stories (What Went Wrong)

    The "Y-Axis Flip" Confusion

    Early versions had watermarks appearing at the top when users dragged to the bottom. The problem: browsers use top-left origin, PDFs use bottom-left origin, and ReportLab uses its own transformation system. 

    // Frontend: 0% from top = user expectation
topPercent = 20  // 20% from top

# Backend: PDF uses bottom-left origin
y = height - (20% * height)  // 20% from top = 80% from bottom

    The "Center vs Corner" Transform

    Center-positioned watermarks needed translate(-50%, -50%) to center at the point, while corner watermarks didn't. The frontend had to track this in dataset and adjust calculations accordingly.

    ❓ Technical Questions People Actually Ask

    "How does ReportLab differ from PyMuPDF for watermarks?"

    ReportLab creates new PDF content from scratch, then PyPDF2 merges it as an overlay. PyMuPDF can modify pages directly. I chose ReportLab because it has better font support and text rendering. The photographer's logo text needed precise font matching - ReportLab delivered.

    "Why use percentages instead of absolute coordinates?"

    Percentages are page-size independent. A watermark at 20% from the top will be at the same relative position on A4, Letter, or any page size. When the writer applied a position from one page to all others in his manuscript, percentages ensured consistency across different page dimensions.

    "How does opacity work in PDFs?" 

    # ReportLab sets RGBA color with alpha
r, g, b = hex_to_rgb(text_color)
c.setFillColor(Color(r/255, g/255, b/255, alpha=opacity))

    PDFs support transparency through the alpha channel in colors. The opacity value (0.0-1.0) is embedded in the PDF and respected by all modern viewers.

    "Can I see the merge_page() code?" 

    # Create watermark overlay as separate PDF
watermark_page = create_watermark_page(...)

# Merge with original page
page.merge_page(watermark_page)
output_writer.add_page(page)

    merge_page() combines two PDF pages into one, preserving all content. It's like layering in Photoshop - the watermark sits on top of the original.

    "What about password-protected PDFs?"

    PyPDF2 can't open encrypted files. Use the Unlock PDF tool first, then add watermarks. The lawyer learned this when his confidential documents were password protected.
