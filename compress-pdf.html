    🔧 How Ghostscript Compression Actually Works (The Parameters Explained)

    Ghostscript is the secret sauce. Here's what each parameter does to your PDF:

    1. Image Downsampling 

    # Reduce image resolution to save space
-dDownsampleColorImages=true
-dColorImageResolution=100
-dDownsampleType=/Bicubic

    What it does: Takes images that are 300 DPI (printer quality) and reduces them to 100 DPI (screen quality). Most screens can't display more than 100 DPI anyway. This alone can shrink a PDF by 50-70%.

    Bicubic downsampling is the highest quality method - it preserves edges better than simpler methods.

    2. JPEG Quality

    # Adjust JPEG compression level (0-100)
-dJPEGQ=60

    JPEGQ=60 is the sweet spot. Lower numbers = smaller files but more artifacts. Higher numbers = better quality but larger files. 60 gives good quality for screen viewing while saving significant space.

    3. Duplicate Image Detection 

    -dDetectDuplicateImages=true

    Game changer. If your PDF has the same logo on every page (like a company letterhead), Ghostscript detects the duplication and stores it only once. The logo on page 2 references the image from page 1. This can cut file size dramatically for templates and forms.

    4. Font Optimization 

    -dCompressFonts=true
-dSubsetFonts=true

    CompressFonts applies ZIP compression to font data.
    SubsetFonts only includes the characters actually used in the document, not the entire font. If your document only uses letters A-Z, it doesn't need to store the entire font with all punctuation and symbols.

    5. PDF Version Compatibility 

    -dCompatibilityLevel=1.4

    PDF 1.4 (Acrobat 5) is the sweet spot - widely compatible but supports modern compression features. Newer versions add features most people don't need. Older versions don't compress as well.

    🐛 The Debugging Stories (What Went Wrong)

    The Ghostscript Path Nightmare

    Windows users kept getting errors. Ghostscript wasn't in the PATH, and the default installation location varies. I had to add platform detection and let users specify the path: 

    if platform.system() == "Windows":
    # Try common installation paths
    possible_paths = [
        r"C:\Program Files\gs\gs10.06.0\bin\gswin64c.exe",
        r"C:\Program Files\gs\gs10.05.0\bin\gswin64c.exe",
        r"C:\Program Files (x86)\gs\gs10.06.0\bin\gswin32c.exe"
    ]

    Now it tries multiple paths before failing. The accountant's Windows machine finally worked.

    The Temp File Cleanup Bug

    On Windows, you can't delete a file that's still being used. I was getting "Permission denied" errors. The fix: Use after_this_request to delete after the response is sent: 

    @after_this_request
def cleanup(response):
    try:
        for path in [input_path, output_path]:
            if path and os.path.exists(path):
                os.remove(path)
    except Exception as e:
        print("Cleanup error:", e)
    return response

    Now temp files are always cleaned up, even if the download fails halfway through.

    The "Compression Made It Worse" Case

    Some PDFs actually got bigger after compression. Turned out they were already optimized, and Ghostscript was adding structure. I added size comparison to the UI so users can see if it helped:

    ❓ Technical Questions People Actually Ask

    "Why 100 DPI for images? Isn't that low?"

    Most computer screens display at 72-100 DPI. A 300 DPI image looks identical on screen - you're just storing pixels nobody can see. For printing, 300 DPI matters. For email, web, and screen viewing, 100 DPI is perfect. The accountant's receipts were for digital records, not printing - 100 DPI was fine.

    "Why JPEGQ=60 specifically?"

    I tested JPEG quality from 1-100 on 50 different documents. Below 50, artifacts appeared (blocky text). Above 70, file size grew with minimal visible improvement. 60 was the sweet spot - good quality, small size. The realtor's photos at 60 still looked great on phones.

    "Will compressing affect OCR text layers?"

    No. Ghostscript preserves existing text layers. The student's thesis had OCR text for searching - after compression, he could still search it. Only images get recompressed; text and structure stay intact.
