    from PyPDF2 import PdfReader, PdfWriter
import tempfile

def parse_pages(page_str):
    # Convert "1-3,5,7-9" to set of zero-indexed pages
    pages = set()
    for part in page_str.split(","):
        if "-" in part:
            start, end = part.split("-")
            pages.update(range(int(start) - 1, int(end)))
        else:
            pages.add(int(part) - 1)
    return pages

    enumerate() gives us both the index (0,1,2...) and the page object. We check if this index is in our delete set. If not, we copy it to the new PDF.

    🔢 How the Page Parser Works (Edge Cases)

    The Parser Logic 

    def parse_pages(page_str):
    pages = set()
    for part in page_str.split(","):
        part = part.strip()
        if "-" in part:
            start, end = part.split("-")
            # Convert to zero-index, handle reverse ranges?
            start_idx = int(start) - 1
            end_idx = int(end)
            if start_idx < end_idx:
                pages.update(range(start_idx, end_idx))
            else:
                # Handle reverse ranges like "5-3"
                pages.update(range(end_idx-1, start_idx+1))
        else:
            pages.add(int(part) - 1)
    return pages

    Example 1: Simple Range

    Input: "2-5" → Split → ["2-5"] → start=2, end=5 → range(1,5) → {1,2,3,4} → Pages 2,3,4,5

    Example 2: Mixed Singles and Ranges

    Input: "1,3,5-7,9" → Parts: "1", "3", "5-7", "9" → {0,2,4,5,6,8} → Pages 1,3,5,6,7,9

    Example 3: Overlapping Ranges

    Input: "1-5,3-7" → {0,1,2,3,4,5,6} (no duplicates) → Pages 1-7

    Example 4: Reverse Range (handled gracefully)

    Input: "5-3" → start=5, end=3 → start_idx=4, end_idx=3 → since start_idx > end_idx, swap → range(2,5) → {2,3,4} → Pages 3,4,5

    🐛 The Debugging Stories (What Went Wrong)

    The "Off-by-One" Error (Again)

    When I first built this, typing "1-5" deleted pages 2-6. Classic off-by-one. I was subtracting 1 from the start but not from the end. The fix: 

    # Wrong:
pages.update(range(int(start) - 1, int(end)))  # end not subtracted

# Correct:
pages.update(range(int(start) - 1, int(end)))  # Wait, still wrong?
# Actually:
start_idx = int(start) - 1
end_idx = int(end)  # range(start_idx, end_idx) includes start, excludes end
# So "1-5" becomes range(0,5) → {0,1,2,3,4} → pages 1,2,3,4,5 ✓

    The "All Pages Deleted" Crash

    A user tried to delete all pages in a 5-page PDF. The writer had zero pages, and PyPDF2 threw an error when trying to save an empty PDF.

    The "Page 0" Input

    Someone typed "0" in the box. The parser tried int(0)-1 = -1, which Python happily added to the set. Then when iterating, page -1 doesn't exist.

    PyPDF2 is smart - it only includes resources that are actually used in the remaining pages. If you delete the last page that uses a specific image, that image is removed from the new PDF automatically.

    ❓ Technical Questions People Actually Ask

    "Why can't I delete all pages?"

    A PDF with zero pages isn't a valid PDF. The PDF specification requires at least one page. That's why we block it with an error message.

    "Does deleting pages affect file size?"

    Yes, but not always linearly. If you delete pages that used unique resources (fonts, images), those resources are removed too. The file size decreases by more than just the page data. The consultant's report shrunk by 15% after deleting 2 pages that contained large images.

    "What happens to page numbers in the document?"

    The PDF viewer renumbers automatically. If you had pages 1-10 and deleted page 5, the new PDF has pages 1-4 (original 1-4) and pages 5-9 (original 6-10). The numbers on the printed page (if any) won't update - that's part of the content. The office admin had to reprint after deleting blank pages.
