How I Built This PDF Editor (Python + PyMuPDF + Fabric.js)

The Problem That Started It All

A few years ago, I was reviewing a contract and needed to make a small change - just a typo fix and a note to the other party. I had to print the document, scribble on it with a pen, scan it back in, and email a messy PDF. It took 15 minutes for what should have been a 30-second edit.

I looked for a free online PDF editor, but everything either wanted a subscription, added watermarks, or required uploading sensitive documents to unknown servers. So I built my own using PyMuPDF for backend processing and Fabric.js for the interactive canvas.

Now I can open any PDF, add text, draw annotations, insert shapes, and save the edited version - all without printing a single page or uploading to the cloud.

def canvas_rect_to_pdf(canvas_x, canvas_y, canvas_w, canvas_h,
                        page_width_pdf, page_height_pdf,
                        canvas_width, canvas_height):
    scale_x = page_width_pdf  / canvas_width
    scale_y = page_height_pdf / canvas_height
    x0 = canvas_x * scale_x
    x1 = (canvas_x + canvas_w) * scale_x
    y0 = page_height_pdf - (canvas_y + canvas_h) * scale_y
    y1 = page_height_pdf - canvas_y * scale_y
    return fitz.Rect(x0, y0, x1, y1)

  2. PDF page: (0,0) at bottom-left, units in points (1/72 inch)

def extract_text_blocks_native(page):
    # Parse PDF's internal text structure
    text_dict = page.get_text("dict")
    for block in text_dict["blocks"]:
        for line in block["lines"]:
            for span in line["spans"]:
                text = span.get("text", "").strip()

                if text and bbox:
                    text_blocks.append({
                        "text": text,

                        "y": bbox[1],
                        "width": bbox[2] - bbox[0],
                        "height": bbox[3] - bbox[1],
                        "fontSize": span.get("size", 12),
                        "fontFamily": map_font_family(span.get("font", "helvetica"))
                    })

2. Font Family Mapping

PDF fonts have weird names like "ABCDEF+Calibri-Bold". The map_font_family() function strips subset prefixes and maps to CSS-safe font names: 

def map_font_family(font_name):
    # Strip subset prefix like "ABCDEF+"
    fn = re.sub(r'^[A-Z]{6}\+', '', font_name)
    
    # Match against known families
    if any(k in fn for k in ['arial', 'arialmt']): return 'Arial'
    elif any(k in fn for k in ['helvetica', 'helv']): return 'Helvetica'
    elif 'times' in fn: return 'Times New Roman'
    else: return 'Arial'

3. Extract Images with Transparency 

def _smask_to_rgba(doc, xref, smask_xref, target_w, target_h):
    # Extract image with soft mask (transparency)
    pix = fitz.Pixmap(doc, xref)
    smask = fitz.Pixmap(doc, smask_xref)
    rgba_pix = fitz.Pixmap(pix, smask)  # Combine with alpha channel
    pil = Image.frombytes("RGBA", (rgba_pix.width, rgba_pix.height),
                          rgba_pix.samples)
    return pil

PDFs store images with optional "soft masks" for transparency. The _smask_to_rgba() function combines the image with its alpha channel to preserve transparency when extracted.

The "Off-By-One" Bullet Point Positioning

When extracting bullet points from PDFs, the bullet character (•) and the following text were separate spans. The text would appear too close to the bullet. The solution was to analyze all spans on the page to find a "standard" bullet text position:

When extracting text, the redaction process would sometimes remove text that wasn't meant to be editable (like watermarks or headers). The fix was to add a text snapshot/restore system:

The Annotation Bar - Dynamic UI Positioning

When you select an object on the canvas, an annotation bar appears above it. This required real-time position calculation: 

function posAnnotBar(fc) {
    const obj = fc.getActiveObject();
    const block = pages[curIdx]?.block;
    const br = block.getBoundingClientRect();
    const ob = obj.getBoundingRect(true, true);
    
    let x = br.left + ob.left + ob.width / 2 - annotBar.offsetWidth / 2;
    let y = br.top + ob.top - annotBar.offsetHeight - 10;
    
    // Keep within viewport bounds
    x = Math.max(6, Math.min(x, window.innerWidth - annotBar.offsetWidth - 6));
    y = Math.max(6, Math.min(y, window.innerHeight - annotBar.offsetHeight - 6));
    
    annotBar.style.left = x + "px";
    annotBar.style.top = y + "px";
}

The bar follows the selected object as it moves, rotates, or scales. Clicking "Edit" opens the appropriate property panel based on object type (text, shape, image, or drawing).

Privacy: How I Built It So I Never See Your Documents

"Do you see my edited documents?" - The lawyer asked this before editing a confidential contract.

1. All Rendering Happens Locally

PDF.js renders pages in your browser. Fabric.js handles all drawing, text, and shape editing on your device. No images are uploaded until you click "Save PDF".

Fabric.js exports paths as SVG commands. The backend parses these and uses PyMuPDF's shape drawing API to recreate them as vector graphics in the PDF.

"How does text extraction handle different fonts?"

PDF fonts have names like "ABCDEF+Calibri-Bold". The map_font_family() function strips the subset prefix and maps to CSS-safe font families. The detect_font_weight_and_style() function checks for "bold", "italic", etc. in the font name to determine styling.

"What's the difference between add_redact_annot() and set_cropbox()?"

add_redact_annot() marks content for permanent removal. When apply_redactions() is called, the content is physically removed from the PDF. set_cropbox() just hides content outside the visible area - it's non-destructive. For the "Edit Content" feature, we use redaction to actually remove the original text so it can be replaced.
