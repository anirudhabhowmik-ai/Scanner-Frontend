    import subprocess
from PyPDF2 import PdfMerger

@excel_to_pdf_bp.route("/excel-to-pdf", methods=["POST"])
def excel_to_pdf():
    files = request.files.getlist("files")
    temp_pdf_paths = []
    
    for file in files:
        temp_id = str(uuid.uuid4())
        input_path = f"/tmp/{temp_id}_{file.filename}"
        file.save(input_path)
        
        # LibreOffice headless conversion
        subprocess.run([
            "soffice",
            "--headless",
            "--convert-to", "pdf",
            "--outdir", "/tmp",
            input_path
        ], check=True)
        
        pdf_path = input_path.rsplit(".", 1)[0] + ".pdf"
        temp_pdf_paths.append(pdf_path)
    
    # Merge if multiple files
    if len(temp_pdf_paths) > 1:
        merger = PdfMerger()
        for pdf in temp_pdf_paths:
            merger.append(pdf)
        merged_path = f"/tmp/{uuid.uuid4()}_merged.pdf"
        merger.write(merged_path)
        merger.close()
        return send_file(merged_path, download_name="converted.pdf")
    
    return send_file(temp_pdf_paths[0], download_name="converted.pdf")

    🔧 How Excel to PDF Conversion Actually Works

    When you upload an Excel file, here's what happens behind the scenes:

    Step 1: File Upload and Save 

    temp_id = str(uuid.uuid4())
input_path = f"/tmp/{temp_id}_{file.filename}"
file.save(input_path)

    Each file gets a unique UUID filename to prevent collisions. Files are saved to /tmp (temporary directory) - automatically cleared on system reboot.

    Step 2: LibreOffice Headless Conversion 

    subprocess.run([
    "soffice",           # LibreOffice executable
    "--headless",        # Run without GUI
    "--convert-to", "pdf",  # Output format
    "--outdir", "/tmp",    # Output directory
    input_path
], check=True)

    --headless is the key. LibreOffice launches in the background, loads the Excel file, and exports it as PDF - exactly as if you'd opened Excel and chosen "Save as PDF". All formatting, charts, and layouts are preserved.

    Step 3: Multiple Sheet Handling

    Excel workbooks with multiple sheets become multi-page PDFs. Each sheet becomes one or more pages, depending on its size and page breaks. My cousin's 12-sheet workbook becomes a 12-page PDF (or more if sheets are long).

    Step 4: PDF Merging (Multiple Files) 

    merger = PdfMerger()
for pdf in temp_pdf_paths:
    merger.append(pdf)
merger.write(merged_path)

    If you upload multiple Excel files, each is converted to PDF, then all are merged in the order you selected. The student's five project files become one PDF, one submission.

    Step 5: Page Size and Orientation

    LibreOffice respects your Excel page setup:

    • Page size - Letter, A4, Legal, etc.
    • Orientation - Portrait or landscape
    • Margins - Exactly as set in Excel
    • Scaling - "Fit to page" settings preserved

    📊 What Gets Preserved (and What Doesn't)

    ✅ Preserved Perfectly

    • Column widths and row heights - Exact pixel-perfect preservation
    • Cell formatting - Colors, borders, number formats, font styles
    • Charts and graphs - Vector quality, fully scalable
    • Images and shapes - Position and size maintained
    • Merged cells - Stay merged
    • Page breaks - Manual and automatic page breaks respected
    • Headers and footers - Print titles, repeating rows
    • Formulas - Display results, not formulas

    ❌ Not Preserved

    • Formulas - PDF shows calculated values only
    • Macros/VBA - Cannot be executed in PDF
    • Data validation - Dropdowns and validation rules
    • Hyperlinks - May not work in all PDF viewers
    • PivotTables - Become static tables
    • Conditional formatting - Results are preserved, rules aren't

    📏 Excel Page Setup - The Most Important Step

    Before converting, check these settings in Excel. They make all the difference:

    1. Page Layout → Orientation

    Portrait for narrow sheets, Landscape for wide sheets. My cousin's sales reports are all landscape now - no more columns getting cut off.

    2. Page Layout → Scale to Fit 

    Width: 1 page
Height: Automatic

    This forces all columns onto one page width. The analyst uses this for his quarterly reports - wide tables fit perfectly.

    3. Page Layout → Print Titles

    Set rows to repeat at top. If your spreadsheet spans multiple pages, the header row repeats on every page. The accountant uses this for client data - column headers on every page.

    4. Page Layout → Breaks

    Insert manual page breaks where you want new pages to start. The HR manager puts each employee's timesheet on a separate page with manual breaks.

    ❓ Technical Questions People Actually Ask

    "Why use LibreOffice instead of Python libraries?"

    Python libraries like openpyxl or pandas can read Excel data, but they can't render the visual layout - column widths, charts, merged cells, print areas. LibreOffice is the actual Excel rendering engine - what you see in Excel is what you get in PDF. My cousin's complex charts would be impossible with pure Python.

    "What's the difference between XLS and XLSX?"

    XLS is the older binary format (Excel 97-2003). XLSX is the newer XML-based format (Excel 2007+). LibreOffice handles both identically - the conversion pipeline is the same. The accountant's clients send both formats, all work fine.

    "Can I see the LibreOffice command that runs?" 

    soffice --headless --convert-to pdf --outdir /tmp /tmp/12345_report.xlsx

    That's it. LibreOffice opens, loads the file, exports as PDF, and exits. All in the background.

    "What about password-protected Excel files?"

    LibreOffice can't open encrypted files without the password. The tool will return an error. My cousin learned this when he password-protected his investor reports - had to remove password before converting.
