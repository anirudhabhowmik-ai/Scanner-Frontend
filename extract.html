    Backend (Python/Flask) 

    from PyPDF2 import PdfReader, PdfWriter
import tempfile

def parse_pages(pages_str, total_pages):
    # Convert "2,5-7,12" to sorted list of zero-indexed pages
    pages = set()
    for part in pages_str.split(","):
        if "-" in part:
            start, end = part.split("-")
            for i in range(int(start)-1, int(end)):
                if 0 <= i < total_pages:
                    pages.add(i)
        else:
            i = int(part) - 1
            if 0 <= i < total_pages:
                pages.add(i)
    return sorted(pages)  # Maintain order for extraction

    🔧 How Page Extraction Actually Works (The Algorithm)

    Extraction is like photocopying - you're creating a brand new document that's a copy of only the pages you want. Here's exactly what happens:

    Step 1: Parse the Page Range 

    # User input: "10,5,2,15-17"
# Step 1: Split on commas → ["10", "5", "2", "15-17"]
# Step 2: Parse each part, convert to zero-index
# Step 3: Keep order, remove duplicates
pages = [9, 4, 1, 14, 15, 16]  # Zero-indexed, in user's order

    Unlike splitting (which creates multiple files), extraction preserves the exact order you specify. If you want page 10 before page 5, that's what you get.

    Step 2: Copy Selected Pages 

    writer = PdfWriter()
for i in pages:
    writer.add_page(reader.pages[i])  # Copy page object

    add_page() doesn't just copy the page content - it copies all dependencies (fonts, images, annotations) that page uses. If two selected pages share a font, PyPDF2 is smart enough to include it only once.

    Step 3: Resource Deduplication 

    Original PDF structure:
├── Page 10 (uses Font-A, Image-X)
├── Page 5 (uses Font-A, Image-X, Image-Y)
├── Page 2 (uses Font-B, Image-Z)

After extraction (pages 10,5,2):
├── Page 10 (uses Font-A, Image-X)
├── Page 5 (uses Font-A, Image-X, Image-Y)
├── Page 2 (uses Font-B, Image-Z)
└── Resources: Font-A (shared), Image-X (shared), Image-Y, Font-B, Image-Z

    PyPDF2 automatically deduplicates shared resources. The extracted PDF is as small as it can be while still containing everything needed.

    Step 4: Write New PDF 

    # Write to temporary file
temp_out = tempfile.NamedTemporaryFile(delete=False, suffix=".pdf")
with open(temp_out.name, "wb") as f:
    writer.write(f)  # PyPDF2 builds complete PDF structure

return send_file(temp_out.name, download_name="extracted_pages.pdf")

    🔄 Extract vs Delete: The Critical Difference (And Why It Matters)

    People mix these up constantly. Here's the technical difference and why you should care:

    📤 Extract (This Tool)

    Creates a NEW PDF containing only the pages you specify. The original file is read, copied, and left completely untouched. It's like making a photocopy of specific pages - the original stays in the folder. 

    # Extract operation - original preserved
reader = PdfReader(original.pdf)  # Read only
writer = PdfWriter()
writer.add_page(reader.pages[4])  # Copy page 5
# original.pdf still exists unchanged

    Use when: You need to share specific pages but keep the full document for yourself. The PhD student extracting methodology section while keeping the full thesis.

    🗑️ Delete (Different Tool)

    Modifies the ORIGINAL (or creates a new version without the deleted pages). Pages are removed permanently (unless you kept a backup). 

    # Delete operation - pages gone
reader = PdfReader(original.pdf)
writer = PdfWriter()
for i, page in enumerate(reader.pages):
    if i not in pages_to_delete:
        writer.add_page(page)  # Skip deleted pages
# New PDF has pages removed

    Use when: You want to permanently clean up a document by removing stuff you don't need.

    🔢 How the Range Parser Works (With Order Preservation)

    The Parser Algorithm 

    def parse_pages(pages_str, total_pages):
    pages = []
    seen = set()
    
    for part in pages_str.split(","):
        if "-" in part:
            start, end = part.split("-")
            for i in range(int(start)-1, int(end)):
                if 0 <= i < total_pages and i not in seen:
                    pages.append(i)
                    seen.add(i)
        else:
            i = int(part) - 1
            if 0 <= i < total_pages and i not in seen:
                pages.append(i)
                seen.add(i)
    
    return pages  # Preserves input order, removes duplicates

    Key features:

    • Order preservation: Pages appear in the output PDF exactly as you typed them
    • Duplicate removal: If you type "5,5,5-5", you get page 5 once
    • Bounds checking: Invalid pages are silently ignored

    Example 1: Reordering Pages

    Input: "10,5,2" → Parser: [9,4,1] → Output PDF: page 10 first, then page 5, then page 2

    Example 2: Mixed Ranges and Singles

    Input: "5-7,2,10-12" → Parser: [4,5,6,1,9,10,11] → Output: pages 5,6,7,2,10,11,12

    Example 3: Duplicates Removed

    Input: "3,3-3,3,5" → Parser: [2,4] → Output: page 3 once, then page 5

    🐛 The Debugging Stories (What Went Wrong)

    The "Set Lost My Order" Bug

    Early versions used a Python set to store pages. Sets are unordered, so when the user typed "10,5,2", they got pages in random order (2,5,10). The designer who needed specific order was furious. 

    # Wrong - unordered
pages = set()
pages.add(9)  # page 10
pages.add(4)  # page 5
pages.add(1)  # page 2
# set iteration could be 1,4,9 or 9,4,1 - unpredictable

# Fixed - use list + seen set
pages = []
seen = set()
if i not in seen:
    pages.append(i)
    seen.add(i)
# pages is now [9,4,1] - exactly as typed

    The "Range End Inclusive" Confusion

    Python's range(start, end) excludes the end. So "5-7" with start=4, end=7 gave pages 5,6 but not 7. Users expected 5,6,7. 

    # Wrong - off by one
for i in range(int(start)-1, int(end)-1):  # end exclusive

# Correct
for i in range(int(start)-1, int(end)):  # includes last page

    The "Page 0" Input

    Someone typed "0" thinking PDFs start at 0. The parser tried i = -1, which Python happily added to the list, then crashed when accessing page -1. 

    # Added bounds checking
if 0 <= i < total_pages:
    pages.append(i)

Extracting pages 45 and 50:
