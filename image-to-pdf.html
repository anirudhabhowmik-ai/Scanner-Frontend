    Backend (Python/Flask + Pillow) 

    from PIL import Image
from io import BytesIO

@image_to_pdf_bp.route("/image-to-pdf", methods=["POST"])
def image_to_pdf():
    images = request.files.getlist("images")
    
    pil_images = []
    for img_file in images:
        # Convert to RGB (PDF doesn't support RGBA)
        img = Image.open(img_file).convert("RGB")
        pil_images.append(img)
    
    pdf_bytes = BytesIO()
    if pil_images:
        first_image, *rest_images = pil_images
        first_image.save(
            pdf_bytes,
            format="PDF",
            save_all=True,
            append_images=rest_images
        )
    
    pdf_bytes.seek(0)
    return send_file(pdf_bytes, download_name="converted.pdf")
    • Pillow (PIL) - Python Imaging Library for image processing
    • convert("RGB") - Handles RGBA transparency by flattening to RGB
    • save_all=True - Creates multi-page PDF from multiple images
    • BytesIO - In-memory file buffer, no disk writes

    🔧 How Image to PDF Conversion Actually Works

    When you upload an image and click convert, here's what happens behind the scenes:

    Step 1: Image Decoding 

    # Pillow automatically detects format (JPG, PNG, GIF, etc.)
img = Image.open(img_file)

    Pillow reads the image headers and decodes the pixel data. It supports over 30 formats automatically.

    Step 2: Color Space Conversion 

    # Convert RGBA (transparent) to RGB (PDF doesn't support alpha)
img = img.convert("RGB")

    Why this matters: PNGs often have transparency (alpha channel). PDFs can't store transparency in the base image layer. The convert("RGB") method flattens the image against a white background. The freelancer's receipts with transparent logos? They become white-backed, clean PDFs.

    Step 3: PDF Generation 

    # Pillow's PDF writer creates a page for each image
first_image.save(
    pdf_bytes,
    format="PDF",
    save_all=True,           # Create multi-page PDF
    append_images=rest_images   # Add remaining pages
)

    The save_all=True parameter tells Pillow to create a multi-page PDF. Each image becomes one page, in the order provided. The property manager's 30 receipts become a 30-page PDF - one click.

    Step 4: Memory Buffer 

    pdf_bytes = BytesIO()
# PDF is written directly to memory, not disk
first_image.save(pdf_bytes, ...)
pdf_bytes.seek(0)  # Rewind for sending
return send_file(pdf_bytes, download_name="converted.pdf")

    BytesIO keeps everything in RAM - no temporary files, no disk writes, instant cleanup. The student's ID photo? Never touched a disk.

    🖼️ How Different Image Formats Are Handled

    JPG / JPEG

    Color space: RGB (no alpha)
    Our handling: Direct conversion, no preprocessing
    Result: Perfect 1:1 quality preservation

    PNG (with transparency)

    Color space: RGBA (Red, Green, Blue, Alpha)
    Our handling: convert("RGB") flattens alpha against white
    Result: Transparent areas become white - perfect for receipts and documents

    GIF (animated)

    Challenge: GIFs can be animated with multiple frames
    Our handling: Pillow reads first frame only
    Result: First frame becomes a single PDF page

    WEBP

    Color space: Can be RGB or RGBA
    Our handling: Same as PNG - convert("RGB")
    Result: Flattened to RGB, quality preserved

    📦 The GIF Surprise

    Someone once uploaded an animated GIF thinking all frames would become PDF pages. Pillow only reads the first frame by default. I added a warning in the FAQ about animated GIFs.

    📏 How PDF Page Sizes Are Determined

    When you convert an image to PDF, the page size matches the image dimensions exactly: 

    # Pillow uses image dimensions for PDF page size
PDF page size = image width (points) × image height (points)
# 1 point = 1/72 inch, so 72 pixels = 1 point

    This means:

    • 1000×800 pixel image → 13.9×11.1 inch PDF page (at 72 DPI)
    • 3000×2000 pixel photo → 41.7×27.8 inch PDF page (huge!)
    • Why it matters: The PDF viewer will scale to fit, but the actual page size matches the image resolution.

    ❓ Technical Questions People Actually Ask

    "Why do I need convert('RGB') for PNGs?"

    PDF's base image format doesn't support transparency. When you call convert('RGB'), Pillow composites the image against a white background. The freelancer's receipt with a transparent logo becomes a white-backed PDF - looks exactly like the printed receipt.

    "Will my GIF animation become multiple PDF pages?" 

    # By default, Pillow reads only first frame
img = Image.open(animated.gif)  # Only frame 1

# To get all frames, you'd need:
while True:
    try:
        img.seek(img.tell() + 1)
    except EOFError:
        break

    We don't currently support animated GIFs as multiple pages. If enough people ask, I'll add it.

    "What DPI are the images in the PDF?"

    PDFs don't have a DPI setting - they store images at their native pixel dimensions. When viewed, the DPI is determined by the display. A 3000×2000 image will look sharp on any screen, regardless of PDF viewer scaling.
