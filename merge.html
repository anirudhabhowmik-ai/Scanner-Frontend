    How Merging PDFs Actually Works

    When you merge PDFs, you're basically telling the computer to take all the pages from File A, then all the pages from File B, and stick them together in order. The formatting, images, and links stay exactly as they were.

    For smaller files (under 20MB): Everything happens right in your browser. Your files never leave your computer. I built it this way specifically so people wouldn't have to worry about privacy.

    For larger files: They get processed on a server because browsers can't handle huge merges. But here's the important part - they're deleted immediately after. I don't keep them, don't look at them, don't store them anywhere.

    Stuff I Learned From Merging Thousands of Files

    • Check the order before merging. I've merged things in the wrong order more times than I'd admit. The list shows you exactly what order they'll combine in. Look at it.
    • Name your files clearly. "scan.pdf" and "scan2.pdf" is a disaster waiting to happen. Name them "contract-page1.pdf" so you know what's what.
    • Big merges can create huge files. If you're merging 50 scanned documents, the result might be 100MB. Use the compress tool afterward if you need to email it.
    • Keep originals. Once you merge, you can't un-merge easily. Keep the original files until you're sure the merged version is what you need.
    • Password-protected PDFs won't work. Unlock them first using the Unlock PDF tool, then merge.

    When NOT to Merge PDFs

    Merging isn't always the answer. Here's when I keep files separate:

    • Different recipients: If you're sending different files to different people, merging makes it worse.
    • Massive files: A 200MB PDF is hard to email, even if it's merged perfectly.
    • Version control: If documents update separately, merging them into one file makes updates messy.
    • Temporary sharing: Sometimes a ZIP folder of separate files is easier for people to navigate.
