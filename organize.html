    🔧 How Drag and Drop Actually Works (The Algorithm)

    The drag-and-drop interface isn't magic - it's careful positioning math. Here's exactly how it works:

    The Drop Position Algorithm 

    function getDragAfterElement(container, y) {
    // Get all cards except the one being dragged
    const draggableElements = [
        ...container.querySelectorAll(".page-card:not(.dragging)")
    ];
    
    // Find the closest element below the drag position
    return draggableElements.reduce((closest, child) => {
        const box = child.getBoundingClientRect();
        const offset = y - box.top - box.height / 2;
        
        if (offset < 0 && offset > closest.offset) {
            return { offset: offset, element: child };
        } else {
            return closest;
        }
    }, { offset: Number.NEGATIVE_INFINITY }).element;
}

    How it works: When dragging, we calculate each card's center point. If the mouse is above a card's center, we insert before it. If below, we insert after. This gives that smooth "slide between" feeling.

    Rendering Pages at Different Scales 

    // Calculate display scale based on screen size
let maxWidth;
if (window.innerWidth < 768) {
    maxWidth = window.innerWidth - 80;  // Mobile: full width minus padding
} else {
    maxWidth = (window.innerWidth - 140) / 2;  // Desktop: half width for 2 columns
}

const displayScale = maxWidth / viewport.width;
const renderScale = displayScale * (window.devicePixelRatio || 2);

    Why two scales? displayScale fits the page in the grid. renderScale multiplies by device pixel ratio to make thumbnails sharp on high-DPI screens (Retina displays). The office manager's 4K monitor gets crisp thumbnails; my old laptop still works fine.

    Memory Management for Large PDFs 

    // Render pages one by one to avoid memory overload
for (let i = 1; i <= pdf.numPages; i++) {
    await renderPage(pdf, i, ...);  // Each page rendered and added to DOM
    // Browser garbage collects between iterations

    Rendering all 500 pages of a document at once would crash the browser. By rendering sequentially and letting the browser clean up, we can handle massive documents. The student's 500-page textbook? Rendered one page at a time, took 30 seconds but worked.

    🔄 How Page Rotation Works (CSS + PyPDF2)

    Frontend: CSS Rotation 

    // Track rotation in data attribute
card.dataset.rotation = newRotation;

// Apply CSS transform
const isVertical = newRotation === 90 || newRotation === 270;
const scale = isVertical ? 0.7 : 1;  // Scale to fit rotated content
contentElement.style.transform = `rotate(${newRotation}deg) scale(${scale})`;

    Why scale? When you rotate a page 90°, its width becomes its height. Without scaling, it would overflow the card. The 0.7 factor keeps it contained while still visible.

    Backend: PyPDF2 Rotation 

    # Apply rotation before adding to new PDF
if rotation != 0:
    page.rotate(rotation)

# For blank pages
blank_page = writer.add_blank_page(width=width, height=height)
if rotation != 0:
    blank_page.rotate(rotation)

    PyPDF2's rotate() modifies the page's /Rotate entry in the PDF dictionary. The viewer reads this and displays the page rotated. The actual content isn't re-encoded - it's just a flag.

    📋 The Layout JSON Format (How Order is Saved)

    Frontend Generates Layout 

    // Convert DOM order to JSON
const layout = [...pagesGrid.children].map(card => {
    if (card.dataset.type === "blank") {
        return {

    🐛 The Debugging Stories (What Went Wrong)

    The "Pages Disappeared" Bug

    Early version lost pages when dragging between positions. Turns out the browser was treating drag-and-drop as a "move" operation that removed the element from the DOM. Had to prevent default behavior: 

    // Prevent browser's default drag behavior
card.addEventListener("dragover", (e) => {
    e.preventDefault();
});

    The "Blank Page Size" Mismatch

    Blank pages were defaulting to A4, but some PDFs were letter size (8.5×11"). The mixed sizes looked terrible when combined. 

    // Fixed: Use first page's dimensions for blank pages
first_page = readers[0].pages[0]
width = float(first_page.mediabox.width)
height = float(first_page.mediabox.height)

    The "Rotation Reset" After Drag

    When dragging a rotated page, the rotation CSS was lost because the browser re-rendered the element. Fixed by storing rotation in data attribute and reapplying after drag: 

    // Store rotation in dataset
card.dataset.rotation = rotation;

// After drag, reapply rotation
const rotation = Number(card.dataset.rotation);
if (rotation) {
    contentElement.style.transform = `rotate(${rotation}deg) scale(0.7)`;
}

    ❓ Technical Questions People Actually Ask

    "Can I organize pages from multiple PDFs at once?"

    Yes. Upload multiple files and all their pages appear together in the grid. The page mapping tracks which file each page came from, so the backend knows where to find them. The consultant does this for proposals - cover from one file, content from another, pricing from a third.

    "How does the drop position calculation work?" 

    // Find the element whose center is closest to the mouse
const afterElement = getDragAfterElement(container, e.clientY);
if (afterElement == null) {
    container.appendChild(draggedElement);
} else {
    container.insertBefore(draggedElement, afterElement);
}

    It checks each card's position and inserts the dragged card before the first card whose center is below the mouse. This gives that smooth "slide between" feeling.

    "What happens to bookmarks and form fields?"

    PyPDF2 preserves them when copying pages. Bookmarks that point to pages are updated to point to the new page numbers. Form fields stay intact. The accountant's Q3 report kept all his margin notes after reorganizing.

    "Why use PDF.js instead of canvas rendering?"

    PDF.js is Mozilla's battle-tested PDF renderer. It handles complex PDFs with fonts, images, and vector graphics correctly. Building our own renderer would be millions of lines of code. PDF.js does it in the browser, for free.

    "Can I see the layout JSON being sent?"

    Open your browser's developer console (F12) and look for the network tab. You'll see the POST request to /organize-pdf with the layout in the request payload. Transparency by design.

    "What about password-protected PDFs?"

    PyPDF2 can't open encrypted files. Use the Unlock PDF tool first, then organize. The consultant learned this when his client proposal was password protected.
