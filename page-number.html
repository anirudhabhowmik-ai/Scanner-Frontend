    🔧 How Page Number Positioning Works (Coordinate Systems)

    The hardest part of this tool was converting between three different coordinate systems. Here's how it works:

    Step 1: Frontend Pixel Coordinates 

    // During drag, we track pixel positions relative to canvas
const canvasRect = canvas.getBoundingClientRect();
const cardRect = card.getBoundingClientRect();
const previewRect = preview.getBoundingClientRect();

// Calculate position relative to canvas
const canvasRelativeX = previewLeft - (canvasRect.left - cardRect.left);
const canvasRelativeY = previewTop - (canvasRect.top - cardRect.top);

    The browser gives us pixel positions relative to the viewport. We convert to positions relative to the canvas (where the page content is).

    Step 2: Convert to Percentages 

    // Convert to percentages for device-independent positioning
const leftPercent = (canvasRelativeX / canvasRect.width) * 100;
const topPercent = ((canvasRect.height - canvasRelativeY) / canvasRect.height) * 100;
// Flip Y so 0% = top, 100% = bottom (matches user expectation)

    Percentages are device-independent - they work whether the page is displayed at 50% zoom or 200% zoom. The backend will convert to actual PDF points.

    Step 3: Backend PDF Points 

    # Convert percentages to PDF points (1 point = 1/72 inch)
center_x_percent = float(left_str) / 100.0
center_y_percent = float(top_str) / 100.0

center_x = center_x_percent * pw  # pw = page width in points
center_y = ph - (center_y_percent * ph)  # Flip Y again for PDF's bottom-left origin

    PDFs use a coordinate system where (0,0) is bottom-left. We flip the Y coordinate again so the number appears where the user dragged it.

    Step 4: Text Baseline Calculation 

    # PyMuPDF inserts text at baseline, not top
font = fitz.Font(fontname=font_name)
asc = font.ascender        # How far above baseline
desc = abs(font.descender) # How far below baseline
text_height = (asc + desc) * font_size
insert_y = center_y + (asc * font_size) - (text_height / 2)

    Fonts have ascenders (parts above baseline) and descenders (parts below). To center text vertically, we calculate where the baseline should be so the text's center aligns with the user's chosen point.

    Browser Viewport Top-left Pixels

    🖱️ The Drag-and-Drop Algorithm (With Boundary Constraints)

    Custom Drag Implementation 

    // Manual drag tracking (not using native HTML5 drag)
const startDrag = (e) => {
    startX = e.clientX;
    startY = e.clientY;
    initialLeft = parseFloat(preview.style.left) || 0;
    initialTop = parseFloat(preview.style.top) || 0;
    
    document.addEventListener("mousemove", drag);
    document.addEventListener("mouseup", stopDrag);
};

const drag = (e) => {
    const dx = e.clientX - startX;
    const dy = e.clientY - startY;
    
    let newLeft = initialLeft + dx;
    let newTop = initialTop + dy;
    
    // Constrain within canvas bounds
    const minX = canvasRect.left - cardRect.left;
    const maxX = minX + canvasRect.width - previewRect.width;
    newLeft = Math.max(minX, Math.min(newLeft, maxX));
    
    preview.style.left = newLeft + "px";
    preview.style.top = newTop + "px";
};

    Native HTML5 drag-and-drop is limited. This custom implementation gives us pixel-perfect control and boundary constraints (can't drag outside the page).

    🐛 The Debugging Stories (What Went Wrong)

    The "Y-Axis Flip" Confusion

    Early versions had page numbers appearing at the top when users dragged to the bottom. The problem: browsers use top-left origin, PDFs use bottom-left origin. 

    // Wrong - no flip
center_y = center_y_percent * ph

// Correct - flip Y
center_y = ph - (center_y_percent * ph)

    The fix: subtract from page height. Now 20% from top in browser = 80% from bottom in PDF = same visual position.

    The "Text Baseline" Nightmare

    Numbers were appearing slightly above or below where users dragged. PyMuPDF inserts text at the baseline, not the top. 

    // Calculate baseline so text center aligns with drag point
asc = font.ascender
desc = abs(font.descender)
text_height = (asc + desc) * font_size
insert_y = center_y + (asc * font_size) - (text_height / 2)

    The "Page Range" Off-by-One

    Users set "Start Page: 2" expecting to skip the cover, but page 1 was still numbered. The problem: zero-indexing vs one-indexing. 

    // Frontend sends 1-based
start_page = parseInt(options.get("startPage", 1))

    ❓ Technical Questions People Actually Ask

    "Why use percentages instead of absolute positions?"

    Percentages are device-independent. If you position a number at 50% from the top on your screen, it will be at 50% from the top on any screen, at any zoom level, and on any paper size when printed. The PhD student's thesis numbers stayed perfectly positioned when printed.

    "What's the difference between PyMuPDF and PyPDF2?"

    PyMuPDF (fitz) can modify page content (add text, draw lines). PyPDF2 can only rearrange pages. For adding numbers, we need PyMuPDF's drawing capabilities. The underline feature uses page.draw_line() which PyPDF2 can't do.

    "How does the underline work?" 

    # PyMuPDF draws lines directly on the page
underline_y = insert_y - 2
page.draw_line(
    (insert_x, underline_y),
    (insert_x + text_width, underline_y),
    color=rgb,
    width=0.5
)

    The line is drawn 2 points below the text baseline, exactly the width of the text. The legal assistant's court filings got proper underlines on numbered pages.
