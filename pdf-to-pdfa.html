    📦 PDF/A Specifications - What Makes It Archival

    PDF/A is an ISO standard (19005) for long-term archiving. Here's what it requires:

    ✅ All fonts must be embedded

    No "font substitution" decades later. The file includes every font it uses.

    ✅ No external dependencies

    No links to external websites, no dependencies on software outside the file.

    ✅ No encryption or passwords

    Archivists must be able to open files forever. No password protection.

    ✅ Device-independent color

    Colors are specified in ways that remain consistent across devices and time.

    ✅ Metadata in XMP format

    Standardized metadata that any system can read.

    ❓ Technical Questions People Actually Ask

    "What's the difference between PDF/A-1b and PDF/A-2b?"

    PDF/A-1b (2005) is the original standard. PDF/A-2b (2011) adds support for transparency, layers, and embedded files. We use PDF/A-2b because it's compatible with modern documents while still being archival. The librarian's newspapers with faded text? PDF/A-2b preserves them better.

    "How does Tesseract's LSTM work?" 

    # Tesseract 4+ uses a two-stage process:
1. CNN (Convolutional Neural Network) detects text regions
2. LSTM (Long Short-Term Memory) recognizes sequences of characters
3. Language model adds context ("c1ear" → "clear")
