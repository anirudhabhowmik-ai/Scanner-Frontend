    🏗️ The Four-Layer Architecture - Why Order Matters

    The secret to perfect PDF to PowerPoint conversion is processing in the right order:

    Layer 1: Tables (FIRST) 

    add_table_layer(slide, page, page_num, scale_x, scale_y)

    Tables must be processed first so they can be detected and stored in `_table_areas`. Later layers check against this list - text inside tables is skipped to avoid duplication.

    Layer 2: Images 

    # Extract raw image by xref to avoid rendering text on top
pix = fitz.Pixmap(page.parent, xref)
img_data = pix.tobytes("png")

    Images are extracted directly from the PDF's xref (cross-reference) to get the raw image without any text rendered on top. This keeps text and images separate.

    Layer 3: Vector Graphics 

    # Convert PDF paths to PowerPoint shapes
paths = page.get_drawings()
for path in paths:
    if should_skip_vector_path(path, rect, page_num):
        continue
    add_powerpoint_shape(slide, path)

    Vector paths (lines, rectangles, freeform shapes) are converted to PowerPoint shapes. Complex paths are rendered as PNG fallback. Background detection removes full-page white rectangles.

    Layer 4: Text (LAST) 

    # Extract text with exact coordinates
blocks = page.get_text("dict")["blocks"]
for line in lines:
    if is_table_area_check(page_num, line_bbox):
        skipped_tbl += 1
        continue
    add_text_box(slide, text, x * scale_x, y * scale_y)

    Text is placed last, with each span at its exact original position. Text inside table areas is skipped (tables already handled in layer 1). This preserves multi-column layouts and bullet points.

    🔤 Font Mapping - From PDF's Weird Names to PowerPoint Fonts

    PDF fonts have names like "ABCDEF+Helvetica-Bold". Here's how we map them: 

    def map_font(pdf_font):
    # Remove subset prefix (ABCDEF+)
    if '+' in pdf_font:
        pdf_font = pdf_font.split('+')[-1]
    
    # Remove style suffixes
    pdf_font = pdf_font.replace('-', '').replace(',', '').lower()
    
    if 'arial' in pdf_font:
        return 'Arial'
    if 'calibri' in pdf_font:
        return 'Calibri'
    if 'times' in pdf_font:
        return 'Times New Roman'
    if 'helvetica' in pdf_font:
        return 'Arial'
    return 'Calibri'

    ❓ Technical Questions People Would Ask

    "Why do you process tables first?"

    Tables need to be detected before text extraction so we know which areas to skip in the text layer. The `_table_areas` cache stores bounding boxes of detected tables. When the text layer runs later, it checks each line against this cache - if a line is inside a table area, it's skipped to avoid duplication. This is why the order matters: Tables → Images → Vectors → Text.

    "How do you extract raw images without text on top?" 

    # Get the raw image by xref (cross-reference), not page rendering
pix = fitz.Pixmap(page.parent, xref)
img_data = pix.tobytes("png")

    Using `page.get_pixmap()` would render the page with all text on top. Using `Pixmap(doc, xref)` extracts the raw image data directly from the PDF's object stream, before any text is rendered on it. This keeps images and text separate, so text can be edited independently.

    "What's the Gemini OCR prompt structure?" 

    '{"text":string,"x0":float,"y0":float,"x1":float,"y1":float,'
 '"font_size_pt":float,"bold":bool}'

    The prompt instructs Gemini to return a JSON array with each text element's exact bounding box (normalized 0-1), font size, and bold flag. This allows precise placement in PowerPoint.
