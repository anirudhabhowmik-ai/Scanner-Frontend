def pdf_to_word(pdf_path, output_path, start_page=0, end_page=None):
    # Extract specific pages if needed

    🔧 How PDF to Word Conversion Actually Works

    Converting PDF to Word is much harder than it sounds. Here's what happens behind the scenes:

    Step 1: Parse PDF Structure

    # pdf2docx reads the PDF's internal structure
- Text objects (characters, their positions, fonts, sizes)
- Images (embedded bitmaps, vectors)
- Paths (lines, rectangles for tables)
- Metadata (author, title, creation date)

    Unlike HTML, PDF doesn't have semantic tags like <h1> or <p>. It just knows where each character sits on the page.

    Step 2: Text Clustering (Finding Paragraphs)

    # Group text by Y-position (same line)
lines = group_by_y(text_objects)

# Group lines by X-position and spacing (same paragraph)
for line in lines:
    if distance_to_previous_line > threshold:
        new_paragraph()
    else:
        add_to_current_paragraph()

    The algorithm figures out which lines belong together. If the gap between lines is small, they're the same paragraph. If it's large, new paragraph.

    Step 3: Font Mapping

    # Map PDF fonts to Word fonts
font_map = {
    "ABCDEF+Helvetica": "Helvetica",
    "TimesNewRomanPSMT": "Times New Roman",
    "CourierNewPSMT": "Courier New"
}

    PDF fonts have weird names like "ABCDEF+Helvetica". pdf2docx strips the prefix and maps to standard Word fonts.

    🔍 OCR for Scanned PDFs - Making Images Editable

    What if your PDF is just scanned images? No text layer at all. That's where OCR comes in: 

    def convert_scanned_pdf(pdf_path, output_path):
    # Step 1: Convert PDF pages to images
    doc = fitz.open(pdf_path)
    images = []
    for page_num in range(len(doc)):
        page = doc[page_num]
        pix = page.get_pixmap(matrix=fitz.Matrix(300/72, 300/72))
        img = Image.frombytes("RGB", [pix.width, pix.height], pix.samples)
        images.append(img)
    
    # Step 2: OCR each image with Tesseract
    text_pages = []
    for img in images:
        text = pytesseract.image_to_string(img, lang="eng")
        text_pages.append(text)
    
    # Step 3: Create Word document with extracted text
    doc = Document()
    for text in text_pages:
        doc.add_paragraph(text)
    doc.save(output_path)

    📋 Why PDF to Word Is Surprisingly Difficult

    PDF and Word are fundamentally different. Here's what makes conversion challenging:

    1. PDF is Fixed Layout, Word is Flowing

    PDF says "put this character at exactly X=100, Y=200". Word says "here's a paragraph, flow it onto the page." Converting between these models is inherently lossy.

    2. No Semantic Information

    PDF doesn't know what's a heading, what's a paragraph, what's a list item. It just knows where text sits. The converter has to infer structure.

    3. Font Embedding vs Font Substitution

    PDFs can embed fonts. Word uses system fonts. When the exact font isn't available, Word substitutes - and layouts shift.

    4. Complex Graphics

    PDF graphics are often vector paths that need to be converted to Word's drawing layer. It's not always perfect.
